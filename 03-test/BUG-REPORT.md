
## Обзор
В процессе тестирования прототипа приложения "Скалодром" было обнаружено и исправлено 3 критических бага, влияющих на основную функциональность приложения.

## BUG-001: Бонусы уходили в отрицательные значения после покупки

### Информация
- **ID:** BUG-001
- **Серьёзность:** Critical
- **Приоритет:** P0
- **Связанный тест-кейс:** TC-03.3
- **Статус:** ✅ Исправлено

### Описание
После первой же покупки с использованием бонусов баланс клиента уходил в отрицательные значения, что позволяло использовать несуществующие бонусы для последующих покупок.

### Шаги воспроизведения
1. Открыть приложение
2. Убедиться, что баланс бонусов = 150
3. Выбрать слот для записи (например, "Вечерний болдеринг" за 1200₽)
4. Выбрать снаряжение "Со своим"
5. Включить чекбокс "💎 Оплатить бонусами"
6. Нажать "Подтвердить запись"
7. Проверить баланс бонусов в шапке

### Ожидаемое поведение
- Баланс бонусов уменьшается на количество использованных бонусов
- Если бонусов недостаточно для полной оплаты, применяется максимальная скидка (но не более 50%)
- Баланс не может стать отрицательным
- Чекбокс "Оплатить бонусами" блокируется при балансе ≤ 0

### Фактическое поведение
- Баланс бонусов становился отрицательным (например, -1050)
- Чекбокс оставался активным даже при отрицательном балансе
- Можно было продолжать использовать "отрицательные" бонусы

### Корень проблемы
В функции `confirmBooking()` при расчёте скидки бонусы списывались в рублях, а не в процентах:
```javascript
// ОШИБКА: bonusUsed рассчитывался в рублях
const bonusUsed = Math.round(totalPrice * discountPercent / 100);
client.bonusBalance -= bonusUsed; // Списываем рубли вместо баллов
```

### Метод исправления
1. **Изменена логика списания бонусов:**
   ```javascript
   // ИСПРАВЛЕНО: bonusUsed = количество баллов (процентов)
   const discountPercent = Math.min(client.bonusBalance, 50);
   const discountRub = Math.round(totalPrice * discountPercent / 100);
   finalPrice = totalPrice - discountRub;
   bonusUsed = discountPercent; // Списываем именно баллы
   client.bonusBalance -= bonusUsed;
   ```

2. **Добавлена блокировка чекбокса при нулевом балансе:**
   ```javascript
   function updateBonusDisplay() {
       if (client.bonusBalance <= 0) {
           checkbox.disabled = true;
           checkbox.checked = false;
           label.classList.add('disabled');
       } else {
           checkbox.disabled = false;
           label.classList.remove('disabled');
       }
   }
   ```

3. **Добавлена проверка перед использованием бонусов:**
   ```javascript
   if (useBonus && client.bonusBalance > 0) {
       // Применяем скидку только если бонусы есть
   }
   ```

### Результат
- ✅ Баланс бонусов корректно уменьшается
- ✅ Баланс не может стать отрицательным
- ✅ Чекбокс блокируется при балансе ≤ 0
- ✅ Скидка применяется корректно (1 балл = 1%, макс 50%)

## BUG-002: Возможность многократной записи на один и тот же слот

### Информация
- **ID:** BUG-002
- **Серьёзность:** Critical
- **Приоритет:** P0
- **Связанный тест-кейс:** TC-02.3
- **Статус:** ✅ Исправлено

### Описание
Клиент мог записаться на один и тот же слот несколько раз, что приводило к дублированию броней и некорректному отображению статуса ("✓ Записан" не появлялся).

### Шаги воспроизведения
1. Открыть приложение
2. Выбрать слот "Вечерний болдеринг" (18:00)
3. Записаться на тренировку
4. Вернуться в расписание
5. Снова нажать "Записаться" на тот же слот
6. Подтвердить запись

### Ожидаемое поведение
- После первой записи кнопка меняется на "✓ Записан" (зелёная)
- Повторная запись на этот слот блокируется
- При попытке записаться появляется сообщение "Вы уже записаны на эту тренировку"

### Фактическое поведение
- Кнопка оставалась "Записаться" после первой записи
- Можно было записаться несколько раз на один слот
- В "Мои записи" появлялись дубликаты
- Баланс бонусов списывался многократно

### Корень проблемы
Отсутствовала проверка наличия активной брони на этот слот перед записью:
```javascript
// ОШИБКА: Нет проверки дублирования
function confirmBooking() {
    // Сразу создаём новую бронь без проверки
    const newBooking = { ... };
    bookings.push(newBooking);
}
```

### Метод исправления
1. **Добавлена проверка перед записью:**
   ```javascript
   function confirmBooking() {
       const isAlreadyBooked = bookings.some(b => 
           b.slotId === currentSlot.id && b.status === 'active'
       );
       
       if (isAlreadyBooked) {
           alert('Вы уже записаны на эту тренировку!');
           return;
       }
       // Продолжаем запись...
   }
   ```

2. **Обновлено отображение кнопок в расписании:**
   ```javascript
   function renderSlots() {
       const isAlreadyBooked = bookings.some(b => 
           b.slotId === s.id && 
           (b.status === 'active' || b.status === 'late_cancel')
       );
       
       if (isAlreadyBooked) {
           actionBtn = `<button class="btn-book already-booked">✓ Записан</button>`;
       } else {
           actionBtn = `<button class="btn-book" onclick="openBooking(${s.id})">Записаться</button>`;
       }
   }
   ```

3. **Добавлена проверка при открытии слота:**
   ```javascript
   function openBooking(id) {
       const isAlreadyBooked = bookings.some(b => 
           b.slotId === id && b.status === 'active'
       );
       
       if (isAlreadyBooked) {
           alert('Вы уже записаны на эту тренировку!');
           return;
       }
       // Открываем экран бронирования...
   }
   ```

### Результат
- ✅ Повторная запись на один слот блокируется
- ✅ Кнопка меняется на "✓ Записан" после записи
- ✅ Нет дубликатов в "Мои записи"
- ✅ Бонусы списываются только один раз

## BUG-003: Невозможность отменить бронь на будущие даты

### Информация
- **ID:** BUG-003
- **Серьёзность:** Critical
- **Приоритет:** P0
- **Связанный тест-кейс:** TC-05.1
- **Статус:** ✅ Исправлено

### Описание
При попытке отменить бронь на следующий день или более поздние даты отмена не работала — статус не менялся, место не освобождалось, уведомление не создавалось.

### Шаги воспроизведения
1. Записаться на тренировку на завтра (или через 2-3 дня)
2. Перейти в "Мои записи"
3. Кликнуть на активную бронь
4. Нажать "Отменить запись"
5. Подтвердить отмену
6. Проверить статус брони и расписание

### Ожидаемое поведение
- Бронь меняет статус на "Отменена"
- Место освобождается в слоте (количество занятых мест уменьшается)
- В расписании кнопка "✓ Записан" меняется обратно на "Записаться"
- Создаётся уведомление "Запись отменена"

### Фактическое поведение
- Статус брони не менялся
- Место не освобождалось
- В расписании оставалось "✓ Записан"
- Уведомление не создавалось
- Функция `executeCancel()` не вызывалась или не работала

### Корень проблемы
1. Отсутствие кнопки отмены для будущих броней
2. Неправильный расчёт времени до старта (учитывалось только время, но не дата)
3. Отсутствие обновления UI после отмены

```javascript
// ОШИБКА: Нет кнопки отмены для будущих броней
function renderBookings() {
    // Для активных броней нет кнопки "Отменить"
    if (b.status === 'active') {
        badge = '<span class="status-badge status-active">Активна</span>';
        // Нет actionBtn для отмены!
    }
}

// ОШИБКА: Неправильный расчёт времени
function showCancelSheet() {
    const now = new Date();
    const startAt = currentBooking.startAt;
    // startAt не содержал дату, только время!
    const hoursUntilStart = (startAt - now) / (1000 * 60 * 60);
}
```

### Метод исправления
1. **Добавлена кнопка отмены для активных броней:**
   ```javascript
   function renderBookings() {
       if (b.status === 'active' && !isPast) {
           badge = '<span class="status-badge status-active">Активна</span>';
           actionBtn = `<button class="btn btn-danger" onclick="openCancelSheet(${b.id})">Отменить запись</button>`;
       }
   }
   ```

2. **Исправлен расчёт времени с учётом даты:**
   ```javascript
   function generateSlotsForDate(dateOffset) {
       // Генерируем полную дату старта (дата + время)
       const startDate = new Date();
       startDate.setDate(startDate.getDate() + dateOffset);
       const [h, m] = slot.time.split(':').map(Number);
       startDate.setHours(h, m, 0, 0);
       
       return {
           // ...
           startAt: startDate, // Полная дата, а не только время
       };
   }
   ```

3. **Добавлено освобождение места при отмене:**
   ```javascript
   function executeCancel() {
       const now = new Date();
       const startAt = currentBooking.startAt;
       const hoursUntilStart = (startAt - now) / (1000 * 60 * 60);
       
       // Находим слот
       const slot = allSlots[currentBooking.slotId % 100][Math.floor(currentBooking.slotId / 100)];
       
       if (hoursUntilStart >= 2) {
           currentBooking.status = 'cancelled';
           // Освобождаем место
           if (slot) {
               slot.booked = Math.max(0, slot.booked - 1);
               if (slot.booked < slot.total) slot.status = 'available';
           }
       } else if (hoursUntilStart > 0) {
           currentBooking.status = 'late_cancel';
           // Место НЕ освобождаем (поздняя отмена)
       }
       
       // Создаём уведомление
       notifications.unshift({
           icon: '❌',
           title: 'Запись отменена',
           text: `${currentBooking.title} (${currentBooking.date})`,
       });
       
       // Обновляем все экраны
       renderBookings();
       renderSlots();
       renderNotifications();
   }
   ```

4. **Добавлено обновление расписания после отмены:**
   ```javascript
   function renderSlots() {
       // Проверяем, есть ли активная бронь на этот слот
       const isAlreadyBooked = bookings.some(b => 
           b.slotId === s.id && 
           (b.status === 'active' || b.status === 'late_cancel')
       );
       
       // Если бронь отменена, кнопка снова "Записаться"
       if (isAlreadyBooked) {
           actionBtn = `<button class="btn-book already-booked">✓ Записан</button>`;
       } else {
           actionBtn = `<button class="btn-book" onclick="openBooking(${s.id})">Записаться</button>`;
       }
   }
   ```

### Результат
- ✅ Отмена работает для броней на любые будущие даты
- ✅ Место освобождается при ранней отмене (≥2ч)
- ✅ Место НЕ освобождается при поздней отмене (<2ч)
- ✅ В расписании кнопка меняется обратно на "Записаться"
- ✅ Создаётся уведомление об отмене
- ✅ Правило 2 часов работает корректно
