# Тестирование приложения "Скалодром"

## Цель
Проверить базовую функциональность клиентского приложения для записи на тренировки.

## Что тестируем
- Запуск приложения
- Просмотр расписания
- Запись на тренировку
- Просмотр своих записей
- Отмена записи

## Приоритеты
- **P0** - Критично (без этого приложение не работает)
- **P1** - Важно (основной функционал)
- **P2** - Желательно (дополнительные проверки)

## Как запускать тесты
1. Открыть `index.html` в браузере
2. Выполнять шаги из тест-кейсов
3. Сравнивать результат с ожидаемым
4. При несовпадении - заводить баг-репорт

## Структура тест-кейсов
- `TC-01-launch.md` - Запуск приложения
- `TC-02-view-schedule.md` - Просмотр расписания
- `TC-03-simple-booking.md` - Простая запись
- `TC-04-view-bookings.md` - Просмотр записей
- `TC-05-simple-cancel.md` - Отмена записи
