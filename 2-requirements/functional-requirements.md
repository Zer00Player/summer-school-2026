### 1. Функциональные требования (ФТ)
*Описывают функции и поведение исключительно клиентского мобильного приложения.*

| ID | Формулировка требования | Приоритет | Источник |
| :--- | :--- | :--- | :--- |
| **ФТ-01** | Отображать список доступных слотов (тренировок) по умолчанию на ближайшие 7 дней от текущего момента. | Высокий | [Бриф], [R-027], [Домен] |
| **ФТ-02** | Предоставлять UI-элементы (фильтр) для выбора дат и просмотра слотов на период более 7 дней. | Средний | [R-027] |
| **ФТ-03** | Отображать empty state («Пока нет доступных тренировок»), если бэкенд вернул пустой список слотов на выбранный период. | Высокий | [R-027] |
| **ФТ-04** | Предоставлять клиенту интерфейс выбора опции снаряжения («со своим» или «нужен прокат») перед подтверждением записи. | Высокий | [Бриф], [Домен] |
| **ФТ-05** | Формировать и отправлять запрос на бронирование в API бэкенда, включая выбранный слот и опцию снаряжения. | Высокий | [Домен], [R-004] |
| **ФТ-06** | Отображать сообщение об отказе в бронировании, если бэкенд возвращает ошибку (например, из-за отсутствия мест, лимита новичков или отсутствия прокатного снаряжения). | Высокий | [R-004], [Домен] |
| **ФТ-07** | Предоставлять клиенту возможность инициировать отмену активной брони через интерфейс приложения. | Высокий | [Бриф], [Домен] |
| **ФТ-08** | Отображать статус брони «Отменена скалодромом» и причину отмены, если эти данные получены от бэкенда. | Высокий | [R-008], [Домен] |
| **ФТ-09** | Блокировать UI-элементы для повторной записи на слот, имеющий статус «Отменена скалодромом». | Высокий | [R-008] |
| **ФТ-10** | Предоставлять форму для отправки оценки инструктору после факта завершения тренировки. | Средний | [Бриф], [Домен] |
| **ФТ-11** | Отображать клиенту входящие push-уведомления (напоминания о записи, факты отмены тренировок скалодромом). | Высокий | [Бриф], [R-008], [Домен] |
