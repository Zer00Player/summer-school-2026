## Обзор

Сквозной паттерн отображения состояний для всех экранов с загрузкой данных:

1. **Loading** — скелетон/шиммер в форме будущего контента
2. **Content** — данные получены и отображены
3. **Empty** — данных нет, заглушка + действие
4. **Error** — ошибка, заглушка + кнопка «Обновить»

### User Story

> Как клиент скалодрома, я хочу всегда понимать, что происходит на экране — данные грузятся, их нет или произошла ошибка.

### Бизнес-ценность

- Воспринимаемая скорость (скелетоны вместо пустого экрана)
- Понятный выход при ошибке
- Единообразие всех экранов

## Состояния

### Таблица состояний

| Состояние | Условие | Отображение |
|-----------|---------|-------------|
| Loading | Ожидание API | Скелетон-загрузка |
| Content | API 200 + данные | Основной контент |
| Empty | API 200 + пусто | Заглушка + действие |
| Error | API 4xx/5xx/сеть | Заглушка + «Обновить» |
| Submitting | Действие в процессе | Лоадер на кнопке |
