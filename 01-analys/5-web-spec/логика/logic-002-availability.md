## Входные данные

| Название | Тип | Описание |
|----------|-----|----------|
| `free_seats` | integer | Свободные места в слоте |
| `capacity_cap` | integer | Потолок мест маршрута (≤8 новички, ≤12 опытные) |
| `free_rental_boards` | integer | Свободные комплекты прокатного снаряжения |
| `GUEST_GROUP_LIMIT` | константа | 3 (себя + до 2 гостей) |
| `seats_count` | состояние | Текущее количество мест |
| `rental_count` | состояние | Количество прокатных комплектов |

---

## Обзор

Логика рассчитывает два независимых лимита:

1. **Лимит мест группы:** `max_seats = min(free_seats, capacity_cap, 3)`
2. **Лимит проката:** `rental_count ≤ free_rental_boards`

«Своё снаряжение» занимает место, но не расходует прокатный фонд. «Прокатное» — занимает место и расходует прокатный фонд.

### User Story

> Как клиент скалодрома, я хочу видеть реальную доступность мест и снаряжения, чтобы выбрать корректное количество мест и не получить отказ при подтверждении.

### Бизнес-ценность

- Предотвращение овербукинга
- Прозрачность для пользователя
- Единый расчёт для всех экранов
