# Мои бронирования

**ID:** SCR-005  
**Тип:** Страница  
**Домен:** 02. Бронирование  
**Приоритет:** Critical  
**Статус:** Черновик  
**Платформа:** Web (Desktop + Mobile)

---

## Обзор

Страница со списком всех бронирований пользователя. Разделена на предстоящие и прошедшие. Позволяет перейти к деталям брони.

### User Story

> Как клиент скалодрома, я хочу видеть все свои записи (предстоящие и прошедшие), чтобы контролировать свои тренировки и при необходимости отменить запись.

## Элементы экрана

### 1. Переключатель вкладок

| Элемент | Описание | Действие |
|:---|:---|:---|
| Вкладка «Предстоящие» | Активные брони | Показать предстоящие |
| Вкладка «Прошедшие» | Прошедшие брони | Показать прошедшие |

### 2. Карточка брони

| Элемент | Описание | Источник данных |
|:---|:---|:---|
| Статус | Бейдж «Активна»/«Отменена» и т.д. | `Booking.status` |
| Дата и время | Полная дата и время | `Slot.start_at` |
| Название маршрута | Название | `Route.name` |
| Тип маршрута | Бейдж «Новички»/«Опытные» | `Route.type` |
| Инструктор | Имя инструктора | `Instructor.name` |
| Количество мест | «2 места» | `seats_count` |
| Снаряжение | «1 прокатная доска» / «своя доска» | `rental_count` |
| Цена | Итоговая цена | `price_total` |
| Карточка целиком | Кликабельная область | Переход на SCR-006 |

### 3. Empty State

| Элемент | Описание |
|:---|:---|
| Иллюстрация | Иконка «пусто» |
| Заголовок | «У вас пока нет записей» |
| Подзаголовок | «Запишитесь на тренировку!» |
| Кнопка «Записаться» | Переход на SCR-001 |

## Критерии приёмки

| ID | Критерий | Приоритет |
|:---|:---|:---|
| AC-001 | **Дано** клиент открывает «Мои записи», **Когда** есть брони, **Тогда** они отображаются в виде списка | P0 |
| AC-002 | **Дано** есть предстоящие и прошедшие брони, **Когда** переключение вкладок, **Тогда** отображаются соответствующие записи | P0 |
| AC-003 | **Дано** клиент кликает по карточке, **Когда** открывается SCR-006, **Тогда** показаны детали брони | P0 |
| AC-004 | **Дано** нет броней, **Когда** открытие страницы, **Тогда** отображается empty state | P0 |
