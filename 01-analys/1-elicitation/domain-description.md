## Описание доменной области (Домен)

На основе брифа можно выделить следующие ключевые сущности и их взаимосвязи в предметной области "Управление записью на скалодром":

### Основные Сущности

1.  **Клиент (Client):**
    *   Пользователь мобильного приложения.
    *   Атрибуты: Имя, контакты (телефон/email), история записей, уровень подготовки (новичок/опытный — *гипотеза, требует уточнения*).
    *   Связи: может иметь много **Броней**.

2.  **Тренировка / Слот (Training Slot):**
    *   Основная единица записи.
    *   Атрибуты: Дата и время старта, длительность (1.5 часа), **Зона/Формат** (болдеринг для новичков, трассы с веревкой), максимальное количество участников (16 для опытных, 8 для новичков), текущее количество свободных мест.
    *   Связи: проводится одним **Инструктором**, включает множество **Броней**.

3.  **Инструктор (Instructor):**
    *   Сотрудник, проводящий тренировку.
    *   Атрибуты: Имя, Рейтинг (на основе оценок клиентов).

4.  **Бронь (Booking):**
    *   Запись конкретного клиента на конкретную тренировку.
    *   Атрибуты: Статус (Активна, Отменена клиентом, Отменена скалодромом), Опция проката снаряжения (Да/Нет), Время создания записи.

5.  **Прокат Снаряжения (Rental):**
    *   Логически связан с бронированием.
    *   Атрибуты: Тип (страховочная система, скальники), Количество единиц (если доступно несколько экземпляров).
    *   *Важно:* Наличие проката влияет на возможность бронирования.

6.  **Оценка / Отзыв (Review):**
    *   Обратная связь от клиента после завершения тренировки.
    *   Атрибуты: Оценка инструктору (числовая/звездная), Текст комментария (опционально), Дата создания.

7.  **Уведомление (Notification):**
    *   Способ коммуникации с клиентом.
    *   Типы: Push-напоминание о предстоящей тренировке, оповещение об отмене тренировки скалодромом.

### Основные Бизнес-Правила (выявленные из брифа):

*   **Лимиты вместимости:** Количество мест в группе зависит от формата (`max_participants` для опытных = 16, для новичков = 8).
*   **Атомарность бронирования:** Гарантия "0 двойных броней" обеспечивается на бэкенде (наша задача — корректно обработать отказ, если мест нет).
*   **Отмена скалодромом:** Слот не удаляется, а меняет статус на "Отменен скалодромом" с обязательным уведомлением клиента. Повторная запись на этот конкретный слот запрещена.
*   **Прокат:** Наличие свободного снаряжения является условием для записи (если клиент его запрашивает).
*   **Горизонт планирования:** По умолчанию показываем расписание на 7 дней вперед (с возможностью фильтрации по датам).

### Контекст системы (Context):

*   **Внешняя система (Backend / Black Box):** Управляет расписанием, инструкторами, зонами, хранит историю. Мы работаем только через API.
*   **Наше приложение (Client App):** Потребляет данные из Backend через API. Отвечает за UI для клиента, навигацию, отображение доступных слотов, процесс бронирования, отмены, оценки инструктора и получение уведомлений.
