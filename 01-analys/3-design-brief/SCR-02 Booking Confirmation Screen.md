# Дизайн-Брифинг: SCR-02 - Экран подтверждения бронирования

## 1. Общая информация
- **ID экрана:** SCR-02
- **Название:** Экран подтверждения бронирования
- **Приоритет:** Высокий
- **Связанные требования:** US-02, ФТ-04, ФТ-05

## 2. Цель экрана
Позволить клиенту выбрать опцию снаряжения (со своим снаряжением или прокат) и подтвердить запись на выбранную тренировку, минимизируя количество шагов до успешного бронирования.

## 3. Целевая аудитория
- Клиенты, уже выбравшие слот и готовые к финальному шагу
- Могут быть как новички (нуждаются в прокате), так и опытные (часто со своим снаряжением)

## 4. Ключевые пользовательские сценарии
1. **Первичный сценарий:** Пользователь выбрал слот → видит детали тренировки → выбирает опцию снаряжения → нажимает "Подтвердить" → получает подтверждение
2. **Альтернативный сценарий:** Пользователь передумал → нажимает "Назад" → возвращается к расписанию
3. **Edge case:** После подтверждения бэкенд возвращает ошибку → переход к экрану отказа (SCR-03)

## 5. Требования к контенту и данным
- **Информация о тренировке:** Дата, время, тип тренировки, инструктор (если доступно), стоимость (если доступна)
- **Опции снаряжения:** 
  - "Со своим снаряжением" (значение: false)
  - "Нужен прокат" (значение: true)
- **Статус проката:** Количество доступного снаряжения влияет на возможность подтверждения (проверяется на бэкенде)
- **Источник данных:** Данные слота переданы с предыдущего экрана; выбор снаряжения формирует payload для API

## 6. Функциональные элементы UI
- **Заголовок экрана:** "Подтверждение записи"
- **Карточка тренировки:** Детальная информация о выбранном слоте
- **Селектор снаряжения:** 
  - Две крупные кнопки/карточки для выбора опции
  - Визуально выделенная выбранная опция
  - Иконки для наглядности (снаряжение/рюкзак)
- **Кнопка подтверждения:** "Записаться" (активна только после выбора опции)
- **Кнопка возврата:** Стрелка "Назад" в левом верхнем углу
- **Примечание:** Краткая информация о правилах проката (если есть)

## 7. Визуальные требования
- **Mobile-first:** Оптимизирован под вертикальный портретный режим
- **Фокус на выборе:** Опции снаряжения занимают центральную часть экрана
- **Цветовая схема:** Акцентный цвет для выделенной опции, нейтральный для невыбранной
- **Крупные кнопки:** Минимум 44px высота, легко нажимаются пальцем
- **Четкие подписи:** "Со своим снаряжением" и "Нужен прокат" без двусмысленности

## 8. Состояния экрана
| Состояние | Описание | Визуальное представление |
|:---|:---|:---|
| **Начальное** | Экран открыт, опция не выбрана | Обе опции активны, кнопка "Записаться" неактивна (серый цвет) |
| **Опция выбрана** | Пользователь выбрал снаряжение | Выбранная опция подсвечена, кнопка активна |
| **Отправка** | Запрос отправляется в API | Индикатор загрузки на кнопке, все UI заблокировано |
| **Успех** | Бронирование подтверждено | Переход на экран активной брони (SCR-04) |
| **Ошибка** | Бэкенд вернул ошибку | Показ сообщения, переход к SCR-03 |

## 9. Взаимодействия и анимации
- **Выбор опции:** Подсветка/увеличение выбранной карточки, плавная смена состояния
- **Переход к успеху:** Анимация успешного подтверждения (галочка) → переход к SCR-04
- **Переход к ошибке:** Показ модального окна или переход к SCR-03
- **Обратная связь на кнопке:** Состояния hover/pressed/disabled

## 10. Варианты дизайна (для рассмотрения)
1. **Вариант A (Радиокнопки):** Классический селектор с радиокнопками и иконками
2. **Вариант B (Карточки выбора):** Две крупные карточки с иконками, текстом и плюшками (например, "Рекомендуем прокат для новичков")
3. **Вариант C (Переключатель):** Toggle-переключатель "Свое снаряжение / Прокат" с пояснениями

**Рекомендуемый вариант:** Вариант B (Карточки выбора) - наиболее наглядный и удобный для быстрого тапа.

## 11. Ограничения и технические нюансы
- **Бизнес-логика:** Проверка доступности прокатного снаряжения на бэкенде - UI только отправляет выбранную опцию
- **Атомарность:** Защита от двойного бронирования обеспечивается бэкендом (при ошибке показываем SCR-03)
- **Отсутствие оплаты:** Экран не содержит полей для оплаты (наличные/перевод вне зоны ответственности приложения)
- **Отмена:** Возврат на SCR-01 возможен через кнопку "Назад"

## 12. Референсы и вдохновение
- Бронирование в сервисах: Booking.com (выбор опций), Airbnb (подтверждение)
- Сервисы проката: CityScoot, Anyday

## 13. Критерии приемки дизайна
- [ ] Информация о тренировке полностью отображается и соответствует выбранному слоту
- [ ] Обе опции снаряжения четко различимы и понятны
- [ ] Выбранная опция имеет визуальное выделение
- [ ] Кнопка "Записаться" неактивна до выбора опции
- [ ] В процессе отправки UI блокируется, отображается индикатор загрузки
- [ ] При успехе корректный переход к SCR-04
- [ ] При ошибке корректный показ отказа (SCR-03)
