# Дизайн-Брифинг: SCR-07 - Экран оценки инструктора

## 1. Общая информация
- **ID экрана:** SCR-07
- **Название:** Экран оценки инструктора
- **Приоритет:** Средний
- **Связанные требования:** US-07, ФТ-10

## 2. Цель экрана
Позволить клиенту оценить инструктора после завершения тренировки, чтобы скалодром мог собирать обратную связь и улучшать качество услуг.

## 3. Целевая аудитория
- Клиенты, посетившие тренировку (завершившие занятие)
- Клиенты, которые получают уведомление с просьбой оценить тренировку

## 4. Ключевые пользовательские сценарии
1. **Первичный сценарий:** Пользователь завершил тренировку → открывает приложение → видит запрос на оценку → выбирает рейтинг и оставляет отзыв → отправляет
2. **Альтернативный сценарий:** Пользователь хочет оценить позже → открывает историю → находит завершенную тренировку → нажимает "Оценить"
3. **Edge case:** Пользователь не хочет оценивать → закрывает экран или нажимает "Пропустить"

## 5. Требования к контенту и данным
- **Информация о тренировке:** Дата, время, имя инструктора, тип тренировки
- **Форма оценки:** 
  - Рейтинг (звезды или шкала) - обязательное поле
  - Текстовый комментарий (опционально)
  - Поля соответствуют контракту API (формат определяется API)
- **Отправка:** Данные отправляются в API после подтверждения

## 6. Функциональные элементы UI
- **Заголовок экрана:** "Оценить тренировку"
- **Карточка тренировки:** Информация для контекста
- **Виджет оценки:** Звезды (1-5) или другой рейтинговый элемент
- **Текстовое поле:** Для комментария (опционально, placeholder: "Поделитесь впечатлениями...")
- **Кнопка отправки:** "Отправить оценку"
- **Кнопка пропуска:** "Пропустить" (возврат на главный)
- **Аватар/имя инструктора:** Фото инструктора (если доступно)

## 7. Визуальные требования
- **Дружелюбный и позитивный дизайн:** Побуждающий к оставлению отзыва
- **Акцент на рейтинге:** Звезды крупные, легко нажимаются
- **Минимализм:** Без лишней информации, фокус на оценке
- **Цветовая схема:** Теплая/позитивная (желтый для звезд, зеленый для кнопки)

## 8. Состояния экрана
| Состояние | Описание | Визуальное представление |
|:---|:---|:---|
| **Начальное** | Экран открыт, оценка не выбрана | Звезды неактивны, кнопка "Отправить" неактивна (серая) |
| **Оценка выбрана** | Пользователь выбрал звезды | Выбранные звезды подсвечены, кнопка активна |
| **Отправка** | Запрос отправляется | Индикатор загрузки на кнопке, UI заблокирован |
| **Успех** | Оценка отправлена | Показ сообщения "Спасибо за отзыв!" → переход на главный |
| **Ошибка** | Ошибка отправки | Сообщение с предложением повторить |

## 9. Взаимодействия и анимации
- **Выбор звезд:** Подсветка звезд при наведении/тапе (анимация)
- **Отправка:** Плавная анимация загрузки на кнопке
- **Успех:** Всплывающее сообщение (toast) или модальное окно с благодарностью
- **Ошибка:** Отображение сообщения об ошибке с кнопкой "Повторить"

## 10. Варианты дизайна (для рассмотрения)
1. **Вариант A (Звезды + Комментарий):** Классический вариант с 5 звездами и полем для текста
2. **Вариант B (Эмодзи + Комментарий):** Оценка через смайлы (😄🙂😐🙁😞) вместо звезд
3. **Вариант C (Слайдер + Комментарий):** Слайдер для оценки (1-10) и поле для текста

**Рекомендуемый вариант:** Вариант A (Звезды + Комментарий) - привычный и понятный пользователям.

## 11. Ограничения и технические нюансы
- **Формат оценки:** Определяется контрактом API (integer, 1-5 или другой)
- **Условие отображения:** Только для завершенных тренировок (по времени)
- **Дублирование:** Приложение не должно позволять повторную отправку для одной тренировки
- **Опциональность:** Комментарий не обязателен, рейтинг - обязателен

## 12. Референсы и вдохновение
- Приложения: Яндекс.Еда (оценка заказа), Uber (оценка поездки), Авито (оценка продавца)
- Сервисы отзывов: Рейтинг в Google Maps, Booking.com

## 13. Критерии приемки дизайна
- [ ] Информация о тренировке и инструкторе отображается корректно
- [ ] Выбор звезд (рейтинга) интуитивно понятен
- [ ] Кнопка "Отправить" активна только после выбора рейтинга
- [ ] Текстовое поле для комментария работает (опционально)
- [ ] При отправке показывается индикатор загрузки
- [ ] При успехе показывается сообщение "Спасибо за отзыв!"
- [ ] При ошибке показывается сообщение с возможностью повторить
- [ ] Кнопка "Пропустить" закрывает экран без отправки

## 14. Блокирующие вопросы к дизайнеру
1. Каким должен быть рейтинг: 5 звезд, 10 звезд или другой формат?
2. Должны ли отображаться уже оставленные оценки в истории?
3. Нужно ли показывать средний рейтинг инструктора рядом с его именем?
4. Как обрабатывать случай, когда пользователь уже оценил тренировку (показывать "Вы уже оценили")?
5. Стоит ли показывать другие тренировки этого же инструктора для сравнения?
