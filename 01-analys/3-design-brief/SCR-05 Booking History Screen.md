# Дизайн-Брифинг: SCR-05 - Экран истории бронирований

## 1. Общая информация
- **ID экрана:** SCR-05
- **Название:** Экран истории бронирований
- **Приоритет:** Средний
- **Связанные требования:** US-05, ФТ-08, ФТ-09

## 2. Цель экрана
Предоставить клиенту доступ ко всем своим бронированиям (активным, прошедшим, отмененным) с возможностью фильтрации по статусу, чтобы можно было отслеживать историю посещений и видеть статус отмененных тренировок.

## 3. Целевая аудитория
- Клиенты, желающие просмотреть историю своих записей
- Клиенты, проверяющие статус отмененной тренировки (отменена скалодромом)

## 4. Ключевые пользовательские сценарии
1. **Первичный сценарий:** Пользователь открывает историю → видит список всех броней в хронологическом порядке → может отфильтровать по статусу
2. **Сценарий проверки отмены:** Пользователь видит, что тренировка отменена → открывает детали → видит причину отмены
3. **Edge case:** История пуста → видит empty state с предложением записаться

## 5. Требования к контенту и данным
- **Список броней:** Все брони клиента, отсортированные по дате (от новых к старым)
- **Статусы:** 
  - "Активна" (будущие)
  - "Завершена" (прошедшие)
  - "Отменена клиентом"
  - "Отменена скалодромом" (с причиной)
- **Фильтры:** По статусу (все, активные, завершенные, отмененные)
- **Детали брони:** Дата, время, тип, инструктор, опция снаряжения
- **Причина отмены:** Для статуса "Отменена скалодромом" - текст причины

## 6. Функциональные элементы UI
- **Заголовок экрана:** "Мои тренировки" или "История"
- **Сегментированный контрол/фильтр:** Вкладки для фильтрации по статусам
- **Карточка брони:** Сокращенная информация с баджиком статуса
- **Клик на карточку:** Переход к деталям брони (SCR-04 или аналогичный)
- **Empty state:** "У вас пока нет записей" с кнопкой "Найти тренировку"
- **Кнопка "Назад":** Возврат на главный экран

## 7. Визуальные требования
- **Группировка по датам:** Карточки сгруппированы по датам (сегодня, завтра, прошедшие...)
- **Статусные баджики:** Цветовая индикация статуса (зеленый - активна, серый - завершена, красный - отменена скалодромом, оранжевый - отменена клиентом)
- **Компактность:** Карточки должны быть информативными, но не громоздкими
- **Фильтр:** Интуитивно понятный (вкладки или дропдаун)

## 8. Состояния экрана
| Состояние | Описание | Визуальное представление |
|:---|:---|:---|
| **Загрузка** | Данные загружаются | Скелетон-загрузка |
| **Есть данные** | Список броней получен | Отображение карточек с группировкой |
| **Нет данных** | История пуста | Empty state с иллюстрацией |
| **Ошибка** | Ошибка загрузки | Сообщение с кнопкой "Повторить" |

## 9. Взаимодействия и анимации
- **Pull-to-refresh:** Обновление списка при свайпе вниз
- **Фильтрация:** Плавное обновление списка при переключении вкладок
- **Переход к деталям:** Анимация перехода на экран деталей
- **Плавный скролл:** Бесконечный скролл для длинных списков

## 10. Варианты дизайна (для рассмотрения)
1. **Вариант A (Вкладки):** Вкладки "Все | Активные | Завершенные | Отмененные"
2. **Вариант B (Дропдаун):** Фильтр в виде выпадающего списка
3. **Вариант C (Календарь + Список):** Календарь вверху, список броней на выбранную дату

**Рекомендуемый вариант:** Вариант A (Вкладки) - наиболее наглядный и быстрый доступ к фильтрации.

## 11. Ограничения и технические нюансы
- **Источник данных:** API бэкенда в реальном времени (без локального кэширования)
- **Актуальность:** При возврате на экран данные обновляются из API
- **Фильтрация:** Фильтрация происходит на стороне приложения (по данным из API) для скорости
- **Сортировка:** От новых к старым (по умолчанию)

## 12. Референсы и вдохновение
- Приложения: Booking.com (история бронирований), Яндекс.Такси (история поездок)
- Спортивные приложения: Strava (активности), MyFitnessPal

## 13. Критерии приемки дизайна
- [ ] Все брони отображаются в хронологическом порядке
- [ ] Статус каждой брони визуально различим (баджики, цвет)
- [ ] Причина отмены скалодромом отображается для соответствующих броней
- [ ] Фильтрация по статусам работает корректно
- [ ] При клике на карточку открываются детали
- [ ] Empty state появляется только при пустом списке
- [ ] Состояние загрузки отображается адекватно
