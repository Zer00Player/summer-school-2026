## Обзор

Страница профиля пользователя. Позволяет просматривать и редактировать имя и телефон, а также выходить из аккаунта.

### User Story

> Как клиент скалодрома, я хочу видеть и изменять свои контактные данные, чтобы быть уверенным, что информация актуальна.

## Элементы экрана

### 1. Блок данных

| Элемент | Описание | Источник данных | Действие |
|:---|:---|:---|:---|
| Имя | Имя пользователя | `Client.name` | Клик по ✎ → редактирование |
| Телефон | Номер телефона | `Client.phone` | Клик по ✎ → редактирование |
| Дата регистрации | Дата | `Client.created_at` | — |

### 2. Редактирование поля

| Элемент | Описание | Действие |
|:---|:---|:---|
| Поле ввода | Инлайн-редактирование | Ввод нового значения |
| Кнопка «✓» | Сохранить | Отправить `PATCH /profile` |
| Кнопка «✕» | Отмена | Вернуться к просмотру |

### 3. Справочные пункты

| Элемент | Описание | Действие |
|:---|:---|:---|
| «Правила клуба» | Ссылка | Открыть внешний URL |
| «Поддержка» | Контакт | Открыть контакт |
| «Версия приложения» | Текст | — |

### 4. Действия с аккаунтом

| Элемент | Описание | Действие |
|:---|:---|:---|
| «Выйти из аккаунта» | Кнопка | Открыть подтверждение → POST /auth/logout |
| «Удалить аккаунт» | Кнопка (деструктивная) | Открыть подтверждение → DELETE /profile |

---

## Используемые запросы

### GET /profile

**Ответ:** Объект `Client`

### PATCH /profile

**Тело запроса:**

| Параметр | Тип | Описание |
|:---|:---|:---|
| `name` | string | Новое имя |

### POST /auth/logout

**Ответ:** 204 No Content

### DELETE /profile

**Ответ:** 204 No Content

## Критерии приёмки

| ID | Критерий | Приоритет |
|:---|:---|:---|
| AC-001 | **Дано** клиент открывает профиль, **Когда** экран загружен, **Тогда** отображаются имя и телефон | P0 |
| AC-002 | **Дано** клиент кликает ✎ у имени, **Когда** вводит новое имя и сохраняет, **Тогда** имя обновляется | P0 |
| AC-003 | **Дано** клиент кликает ✎ у телефона, **Когда** вводит новый номер и сохраняет, **Тогда** телефон обновляется (с подтверждением кода) | P0 |
| AC-004 | **Дано** клиент кликает «Выйти», **Когда** подтверждает, **Тогда** происходит выход | P0 |
