# Подтверждение отмены

**ID:** BS-003  
**Тип:** Модальное окно  
**Домен:** 02. Бронирование  
**Приоритет:** High  
**Статус:** Черновик  
**Платформа:** Web (Desktop + Mobile)

---

## Обзор

Модальное окно подтверждения отмены записи. Объясняет последствия отмены (ранняя/поздняя) и требует подтверждения.

### User Story

> Как клиент скалодрома, я хочу подтвердить отмену записи, понимая последствия, чтобы случайно не потерять место.

## Элементы экрана

### 1. Заголовок

| Элемент | Описание |
|:---|:---|
| Иконка | ❓ |
| Заголовок | «Отменить запись?» |

### 2. Правило 2 часов

| Элемент | Описание |
|:---|:---|
| Текст | «Отмена не позднее чем за 2 часа до старта — место освобождается. Позже — место остаётся за вами, но штрафов нет.» |

### 3. Блок текущего случая

| Состояние | Отображение |
|:---|:---|
| Ранняя (≥ 2ч) | «Места возвращаются в слот и станут доступны другим.» |
| Поздняя (< 2ч) | «Поздняя отмена: место не освобождено. Штраф не взимается.» |

### 4. Кнопки

| Элемент | Описание | Действие |
|:---|:---|:---|
| «Подтвердить отмену» | Деструктивная | POST /bookings/{id}/cancel |
| «Не отменять» | Безопасная | Закрыть модальное окно |

---

## Используемые запросы

### POST /bookings/{bookingId}/cancel

**Обработка ответа:**

| Код | Сценарий | Действие |
|:---|:---|:---|
| 200 | Успех (ранняя/поздняя) | Закрыть → обновить SCR-006 |
| 422 | Слот стартовал | Сообщение об ошибке |
| 409 | Уже отменена | Актуализировать статус |

## Критерии приёмки

| ID | Критерий | Приоритет |
|:---|:---|:---|
| AC-001 | **Дано** до старта ≥2ч, **Когда** открывается модальное окно, **Тогда** показана ранняя отмена | P0 |
| AC-002 | **Дано** до старта <2ч, **Когда** открывается модальное окно, **Тогда** показана поздняя отмена | P0 |
| AC-003 | **Дано** клиент подтверждает отмену, **Когда** запрос успешен, **Тогда** бронь отменяется | P0 |
| AC-004 | **Дано** клиент нажимает «Не отменять», **Когда** модальное окно закрывается, **Тогда** бронь сохраняется | P0 |
