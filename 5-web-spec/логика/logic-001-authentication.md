## Входные данные

| Название | Тип | Возможные значения | Описание |
|----------|-----|-------------------|----------|
| `access_token` | Хранилище (localStorage / sessionStorage) | JWT-строка / отсутствует | Bearer-токен доступа. Подставляется в `Authorization: Bearer <access_token>` во все авторизованные запросы. |
| `refresh_token` | Хранилище (localStorage / sessionStorage) | строка / отсутствует | Refresh-токен для обновления access-токена. |
| `expires_in` | Состояние | integer (секунды) | Срок жизни access-токена. |
| `phone` | Состояние | E.164, `^\+7[0-9]{10}$` | Номер телефона из шага 1. |
| `is_new` | Состояние | `true` / `false` | Признак нового пользователя из ответа `verifyAuthCode`. |

## Обзор

Логика описывает вход в веб-приложение по номеру телефона без пароля через одноразовый код (OTP). Применяется на странице входа (SCR-001) и реализует трёхшаговый поток:

1. Ввод телефона
2. Ввод кода из SMS
3. Ввод имени (только для нового пользователя)

После подтверждения кода сервер выдаёт пару JWT-токенов (access + refresh), которые сохраняются в localStorage и обеспечивают автоматический вход при последующих визитах.

### User Story

> Как клиент скалодрома, я хочу входить в систему по номеру телефона без пароля, чтобы быстро записаться на тренировку.

### Бизнес-ценность

- Минимальный порог входа — нет пароля
- Сохранённая сессия пропускает экран входа
- Безопасность: токены хранятся в localStorage
