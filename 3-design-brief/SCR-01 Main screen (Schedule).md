# Дизайн-Брифинг: SCR-01 - Главный экран (Расписание)

## 1. Общая информация
- **ID экрана:** SCR-01
- **Название:** Главный экран (Расписание)
- **Приоритет:** Высокий
- **Связанные требования:** US-01, ФТ-01, ФТ-02, ФТ-03

## 2. Цель экрана
Предоставить клиенту быстрый и понятный доступ к расписанию тренировок на ближайшие 7 дней с возможностью выбора слота для записи.

## 3. Целевая аудитория
- Клиенты скалодрома, использующие мобильное приложение преимущественно в зале или на ходу
- Возраст: 16-45 лет
- Сценарий: быстрый просмотр доступных тренировок и запись в пару касаний

## 4. Ключевые пользовательские сценарии
1. **Первичный сценарий:** Пользователь открывает приложение → видит расписание на неделю → выбирает подходящий слот → переходит к бронированию
2. **Альтернативный сценарий:** Пользователю нужна дата за пределами 7 дней → использует фильтр → просматривает слоты на выбранную дату
3. **Edge case:** На выбранный период нет доступных тренировок → видит empty state с рекомендацией проверить другие даты

## 5. Требования к контенту и данным
- **Отображение слотов:** Список доступных тренировок на ближайшие 7 дней от текущей даты (по умолчанию)
- **Информация о слоге:** Дата, время, тип тренировки (новичковая/опытная), количество свободных мест (из 8 или 16), статус (доступно/занято/отменено)
- **Фильтр:** UI-элемент для выбора конкретной даты или периода (неделя/месяц)
- **Empty state:** Текст "Пока нет доступных тренировок" с кнопкой "Проверить другие даты"
- **Источник данных:** API бэкенда в реальном времени (без локального кэширования)

## 6. Функциональные элементы UI
- **Заголовок экрана:** "Расписание" или "Тренировки"
- **Навигация:** Фильтр дат (свайп или календарь)
- **Карточка слота:** Каждая тренировка в виде карточки с:
  - Датой и временем
  - Названием/типом тренировки
  - Индикатором свободных мест (прогресс-бар или числом)
  - Кнопкой "Записаться" (активна, если есть места)
- **Статусы слота:** Визуальное различие между доступными, занятыми и отмененными слотами
- **Empty state:** Центрированное сообщение с иллюстрацией

## 7. Визуальные требования
- **Mobile-first:** Дизайн адаптирован под экраны смартфонов (375px - 428px ширина)
- **Иерархия:** Самые ближайшие тренировки сверху, затем по возрастанию даты
- **Цветовая схема:** Контрастная, в стилистике бренда скалодрома
- **Типографика:** Четкая, хорошо читаемая в условиях зала (возможное яркое освещение)
- **Интерактивные элементы:** Крупные кнопки (минимум 44px высота), удобные для тапа

## 8. Состояния экрана
| Состояние | Описание | Визуальное представление |
|:---|:---|:---|
| **Загрузка** | Данные загружаются с бэкенда | Скелетон-загрузка (placeholder) |
| **Данные есть** | Список слотов получен | Отображение карточек тренировок |
| **Нет данных** | Пустой список от бэкенда | Empty state с предложением проверить другие даты |
| **Ошибка загрузки** | Ошибка сети или API | Сообщение с кнопкой "Повторить" |

## 9. Взаимодействия и анимации
- **Pull-to-refresh:** Обновление расписания при свайпе вниз
- **Плавный скролл:** Бесконечный скролл для длинных списков (виртуализация)
- **Переход к бронированию:** Анимация перехода к экрану подтверждения (SCR-02) при нажатии на "Записаться"
- **Фильтр дат:** Свайп по датам или вызов календаря
- **Обратная связь:** Визуальная подсветка нажатия на карточку

## 10. Варианты дизайна (для рассмотрения)
1. **Вариант A (Список):** Классический вертикальный список карточек с полной информацией
2. **Вариант B (Сетка):** Двухколоночная сетка с карточками для быстрого сканирования
3. **Вариант C (Календарь + Список):** Миникалендарь в верхней части + список на выбранную дату

**Рекомендуемый вариант:** Вариант A (Список) - обеспечивает максимальную информативность и удобство чтения в условиях зала.

## 11. Ограничения и технические нюансы
- **Бизнес-логика:** Вся логика проверки доступности на бэкенде - UI только отображает данные и не должен дублировать проверки
- **Реальное время:** Данные всегда актуальные с бэкенда (нет кэширования)
- **7-дневный лимит:** По умолчанию показываются только 7 дней, но фильтр позволяет смотреть дальше
- **Отмененные слоты:** Слоты со статусом "Отменена скалодромом" должны быть визуально недоступны для записи

## 12. Референсы и вдохновение
- Приложения для бронирования: ClassPass, Mindbody
- Спортивные приложения: Rock Gym Pro, MyClimb
- Календарные приложения: Google Calendar, Apple Calendar

## 13. Критерии приемки дизайна
- [ ] Все карточки слотов отображают дату, время, тип и свободные места
- [ ] UI четко различает доступные, занятые и отмененные слоты
- [ ] Кнопка "Записаться" неактивна/скрыта для недоступных слотов
- [ ] Фильтр дат интуитивно понятен и доступен
- [ ] Empty state появляется только при пустом ответе API
- [ ] Состояние загрузки отображается адекватно (скелетон или индикатор)
- [ ] Дизайн адаптивен под различные размеры экранов (Mobile-first)
