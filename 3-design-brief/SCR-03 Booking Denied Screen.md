# Дизайн-Брифинг: SCR-03 - Экран отказа в бронировании

## 1. Общая информация
- **ID экрана:** SCR-03
- **Название:** Экран отказа в бронировании
- **Приоритет:** Высокий
- **Связанные требования:** US-03, ФТ-06

## 2. Цель экрана
Предоставить клиенту понятное и дружелюбное сообщение об отказе в бронировании с четким объяснением причины, чтобы он мог быстро переориентироваться и выбрать другой слот.

## 3. Целевая аудитория
- Клиенты, пытавшиеся записаться на тренировку, но получившие отказ
- Важно: состояние фрустрации - нужна эмпатичная коммуникация

## 4. Ключевые пользовательские сценарии
1. **Первичный сценарий:** Пользователь подтвердил запись → бэкенд вернул ошибку → приложение показывает понятный отказ → пользователь возвращается к расписанию
2. **Причины отказа:**
   - "Нет свободных мест" (лимит достигнут)
   - "Нет прокатного снаряжения" (фонд закончился)
   - "Недоступно для новичков" (превышен лимит новичков)
   - "Техническая ошибка" (общее сообщение)

## 5. Требования к контенту и данным
- **Сообщение об ошибке:** Текст отказа, сформированный на основе кода ответа API
- **Причина отказа:** Конкретная причина (для каждой ошибки свой текст)
- **Рекомендация:** Предложение альтернативных действий (проверить другие слоты)
- **Дополнительная информация:** Если доступно - количество свободных мест на соседние слоты

## 6. Функциональные элементы UI
- **Иконка отказа:** Визуальный индикатор проблемы (например, красный крест, часы, закрытый календарь)
- **Заголовок отказа:** "Не удалось записаться" или аналогичный
- **Сообщение о причине:** Детальное объяснение
- **Кнопка действия:** "Показать другие слоты" → возврат на SCR-01 (с обновлением даты)
- **Кнопка возврата:** "Назад" (возврат на SCR-02 с сохранением выбора снаряжения)
- **Альтернативные действия:** Возможно, предложение записаться на другой день/время

## 7. Визуальные требования
- **Дружелюбный дизайн:** Даже при ошибке - мягкие цвета, понятные иконки
- **Информативность:** Причина отказа видна сразу, без скролла
- **Контрастность:** Текст хорошо читается на фоне
- **Акцентная кнопка:** Основное действие (показать другие слоты) - яркая, привлекающая внимание
- **Негативный сценарий:** Не использовать критические красные цвета (тревога), лучше оранжевый/желтый для информирования

## 8. Состояния экрана
| Состояние | Описание | Визуальное представление |
|:---|:---|:---|
| **Отказ по местам** | "Нет свободных мест" | Иконка "полный зал", текст с извинениями |
| **Отказ по прокату** | "Нет прокатного снаряжения" | Иконка "снаряжение недоступно", предложение прийти со своим |
| **Отказ по лимиту новичков** | "Превышен лимит новичков" | Иконка "новичок", предложение выбрать другую тренировку |
| **Техническая ошибка** | Общий отказ | Стандартное сообщение с предложением попробовать позже |

## 9. Взаимодействия и анимации
- **Появление экрана:** Плавное появление из нижней части экрана (sheet) или полноэкранный переход
- **Кнопка "Показать другие слоты":** При нажатии закрывает экран и возвращает на SCR-01 (возможно, с обновленными данными)
- **Кнопка "Назад":** Возврат к SCR-02
- **Дизмисс:** Возможность свайпнуть вниз для закрытия (если реализован как sheet)

## 10. Варианты дизайна (для рассмотрения)
1. **Вариант A (Bottom Sheet):** Отказ показывается как модальное окно снизу (не блокирует полностью расписание)
2. **Вариант B (Полноэкранный):** Отказ занимает весь экран с четким сообщением и действиями
3. **Вариант C (Toast + переход):** Показ тост-уведомления и автоматический возврат на SCR-01 с выделением отказа

**Рекомендуемый вариант:** Вариант A (Bottom Sheet) - не вырывает пользователя из контекста, позволяет быстро вернуться к выбору.

## 11. Ограничения и технические нюансы
- **Текст ошибки:** Формируется на основе кода ответа API (не жестко зашит в приложение)
- **Нет кэширования:** При возврате на SCR-01 данные обновляются из API
- **Отсутствие повторных попыток:** Приложение не пытается автоматически повторить запрос (чтобы избежать дублирования)
- **Сохранение выбора:** При возврате на SCR-02 выбранная опция снаряжения сохраняется

## 12. Референсы и вдохновение
- Сервисы бронирования: Booking.com (отказ при недоступности), Aviasales (нет мест)
- Приложения: Яндекс.Еда (ошибка заказа), Uber (нет машин)

## 13. Критерии приемки дизайна
- [ ] Отображается понятный заголовок отказа
- [ ] Причина отказа четко сформулирована и соответствует коду ошибки
- [ ] Кнопка "Показать другие слоты" работает и возвращает к расписанию
- [ ] Кнопка "Назад" работает и возвращает к SCR-02
- [ ] Дизайн адаптирован под разные типы отказов (цвета, иконки)
- [ ] Не используется чрезмерно тревожная цветовая гамма (красный только для акцентов)
